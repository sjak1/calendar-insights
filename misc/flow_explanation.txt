SESSION ID FLOW

1. UI CREATES SESSION ID (index.html lines 301-308)
   - getSessionId() checks localStorage for 'chat_session_id'
   - If missing, creates: 'session_' + timestamp + random string
   - Stores in localStorage

2. UI SENDS TO API (index.html line 407)
   - POST /process_query with body: {query, headers: {}, session_id}

3. API RECEIVES (api.py line 38)
   - Extracts session_id from QueryPayload
   - Passes to handle_query() (line 41)

4. API RETURNS (api.py line 43)
   - Response: {"message": result, "session_id": session_id}

5. UI UPDATES (index.html lines 437-439)
   - If server returns different session_id, updates localStorage


UI TO API FLOW

1. USER ACTION
   - Types query in input field
   - Clicks Send or presses Enter

2. FRONTEND (index.html sendQuery function, line 379)
   - Gets session_id from localStorage
   - Shows loading indicator
   - POST to /process_query with:
     * query: user's text
     * headers: {} (empty)
     * session_id: from localStorage

3. API ENDPOINT (api.py /process_query, line 34)
   - Receives QueryPayload
   - Extracts: query, headers, session_id
   - Calls handle_query(query, headers, session_id=session_id)

4. RESPONSE HANDLING
   - API returns: {"message": result, "session_id": session_id}
   - UI removes loading indicator
   - Displays message in chat
   - Updates session_id if changed


UVICORN COMMAND BREAKDOWN

Command: uvicorn api:app --reload

1. api:app
   - api = module name (api.py file)
   - app = FastAPI instance (line 13 in api.py)

2. --reload
   - Auto-restarts server on file changes
   - Watches project directory

3. SERVER STARTUP
   - Starts on http://127.0.0.1:8000
   - Mounts /static directory (serves static files)
   - Root endpoint (/) serves index.html
   - /process_query endpoint handles POST requests

4. STATIC FILES
   - /static/* serves files from static/ directory
   - index.html served at root (/)
