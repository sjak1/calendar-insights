🎯 How It Works:

  1. User asks: "Show me all COO presenters" (natural language)
  2. LlamaIndex receives: Query goes to SQLTableRetrieverQueryEngine
  3. Vector search: Embeds query and searches 7 table schemas for relevance
  4. Table selection: Picks most relevant table (e.g., t_request_agenda_presenter)
  5. SQL generation: GPT reads table context + Oracle rules → generates SQL
  6. SQL modification: Strips semicolons, fixes Oracle-specific syntax
  7. Execute: Runs query on Oracle database (AWS RDS)
  8. Get results: Raw data returned from database
  9. Format response: GPT converts data to natural language
  10. Return: User gets readable answer

⚠️ Oracle Database Special Rules

🔴 Critical Rules (Enforced by System):

  • NO SEMICOLONS: Oracle driver fails with ; at end of SQL
  • Use FETCH FIRST: Not LIMIT/OFFSET (e.g., FETCH FIRST 10 ROWS ONLY)
  • UPPERCASE tables: Table/column names are UPPERCASE unless quoted
  • NVL not COALESCE: Use NVL(expr1, expr2) for NULL handling
  • CLOUD_DATE UDT: Use .ZONEDATE attribute for date columns
  • LIKE for custom fields: text_field_3 contains formatted strings, use LIKE patterns

💡 Smart Search Strategy:

  • Standard columns: Use exact matches (e.g., first_name = 'John')
  • Custom fields: Use LIKE patterns (e.g., text_field_3 LIKE '%COO%')
  • Job titles: Stored as "CIO - Chief Information Officer", search with abbreviation
  • Dietary: JSON arrays like ["Halal"], use LIKE '%Halal%'
