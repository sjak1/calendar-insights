Presenter Suggestion — End-to-End Flow

What happens when the agenda generator asks "who should present at this meeting?"

Step 1 — Inputs

What we know about the meeting

The agenda generator passes these values into the presenter engine:

event_id customer_name industry topic

At least one must be present. Three separate calls are made — one per "scope" — ranked from most-specific to least-specific.

Scope 1 — Same Event

event_id only. "Who presented at this exact event before?"

Scope 2 — Same Company

customer_name only. "Who presented for this company before?"

Scope 3 — Same Industry

industry only. "Who presents on topics for this industry?"

Step 2 — Scope Resolution (events index)

customer_name / industry → list of event_ids

We can't filter activities by company name directly — so we first query the events index to find matching event IDs.

eventData.VISIT_INFO.data.customerName eventData.VISIT_INFO.data.customerIndustry → event_ids[] 
{
  "query": {
    "bool": {
      "should": [
        { "term":  { "eventData.VISIT_INFO.data.customerName.keyword": "Slack" }},
        { "match": { "eventData.VISIT_INFO.data.customerName": { "query": "Slack", "fuzziness": "AUTO" }}},
        { "term":  { "eventData.VISIT_INFO.data.customerIndustry.keyword": "Technology" }}
      ],
      "minimum_should_match": 1
    }
  },
  "size": 50
}

Returns up to 50 event IDs — passed to Step 3 as a filter.

Step 3 — Activity Search (activities index)

Find activities that have real presenters attached

activities

Filter: must have a presenter with an email. Optionally filter by event_ids. Topic adds a relevance boost but is not required.

activityInfo.topic_presenter.data.presenter.primaryEmail eventId.keyword activityInfo.topic.data.topic.textField1 startTime.utcMs 
{
  "query": {
    "bool": {
      "must": [
        { "exists": { "field": "activityInfo.topic_presenter.data.presenter.primaryEmail" }},
        { "terms": { "eventId.keyword": ["EVT-001", "EVT-002", ...] }}
      ],
      "should": [
        { "match": { "activityInfo.topic.data.topic.textField1": { "query": "Cloud Migration", "boost": 3 }}}
      ]
    }
  },
  "size": 100,
  "sort": [{ "startTime.utcMs": { "order": "desc" }}]
}
Fallback: If this returns 0 hits (e.g. the event is new with no history), we drop all filters and fetch the 200 most recent activities globally that have presenter emails.
Step 4 — Extract Presenters from Hits

Each activity has a topic_presenter array

We loop over every element in the array per activity hit and read these fields:

data.presenter.firstName data.presenter.lastName data.presenter.primaryEmail data.presenter.designation data.presenterStatus

Skipped

presenterStatus == "declined", "rejected", or "cancelled"

Keyed on

primaryEmail (lowercase). Same person across events is deduplicated.

Tracked per presenter

session_count, accepted_count, list of topics, list of event_ids, latest timestamp

Step 5 — Rank & Return

Sort by quality signals, return top N

#SignalWhy
1accepted_count ↓Confirmed sessions outweigh tentative ones
2session_count ↓More sessions = more experienced presenter
3event_count ↓Breadth — presented at more distinct events
4latest_ts ↓Recency — active presenters preferred
